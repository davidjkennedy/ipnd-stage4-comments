Lesson 1: The Web, HTML & CSS

How the Web Works

The World Wide Web was created in the early nineties and has become a system supporting the sharing, storage, linking and displaying of specially formatted text documents using the markup language of HTML.

Hyper Text Markup Language or HTML is made up of text, markup tags, and reference links to other pages and media (images, video, audio). Markup extends text into html making it richer. Internet based HTML documents stored on servers can be displayed on a computer that has an internet connection by pointing a Web Browser to a unique internet address otherwise known as a URL. The rules for the tranfer and storage of these files from server to browser is called HTTP.

The syntax of code is strict, and there are many rules, too many to memorize, if you forget something you can always look it up quickly at the W3 website but if you obey the rules you can create amazing things!.

Tags and Elements

A HTML Element is made up of the content within a set of opening and closing tags. An Elements attributes are always specified at the start of a tag and provides additional info about the element.

Opening Tag:
All tags are contained within brackets i.e. the paragraph opening tag.
Content:
whatever the content happens to be.
Closing Tag:
the paragraph closing tag.

Note: some tags (void tags) do not require a closing tags. e.g. the line break tag <br>.

White Space, Block, and Inline Elements In HTML

Whitespace is always just one space, so to create space around elements/text use tags. e.g. the line break tag.<br>
Inline Elements are normally displayed without line breaks.
Examples of Inline tags: <span>.
The span tag is used to group inline elements together for CSS styling, creates no change on its own. More detail on span later ... it should be closed like this:</span>.
Block Level Elements are normally start with a new line.
Examples of Block tags: <div>.
The div tag is used to group block elements together for CSS styling, by default browsers always places a line break before and after the div element. More detail later ... it should be closed like this: </div>.

More Tags

The br tag <br> is used to insert a single line break in content, it is a void tag so it doesn't need to be closed except for in xhtml where it should be closed like this:</br>.
The tag <p> by default browsers will automatically add a margin before and after the paragraph element.

Lesson 1: Intro to Serious Programming

The Computer

A Computer is a universal machine that can be programmed to emulate anything. A Computer Program is a sequence of explicit instructions which the computer can understand. Procedural Thinking is needed to generate logical instructions in a computer program. Technological Empathy is understanding the technology ecosystem and how it all works.

Programming and Python

Programming is at the heart of Computer Science. Natural Language is Ambiguous and Verbose. Computer programs need to be written as a logical succinct sequence of commands. In Python for instance you type code in a Python interpreter (text editor e.g. Sublime). The grammar in program languages is strict and errors can cause problems (syntax errors stopping the program from running) or worse not giving an error message but causing incorrect or unpredictable results. When the code is run the interpreter compiles it into Binary machine readable code that the computer understand directly.

Computer Programming Grammar

John Backus developed Backus–Naur Form when leading the team that developed FORTRAN (the first widely used high level programming language). Its the basis for programming languages, with a very simple and concise grammar. The syntax is derived through the process of derivation to a point where all non-terminals are replaced until only terminals are left. Terminals being the elementary symbols of the language defined by its formal grammar.

Lesson 3: Input - Function - Output

What is a Function

A Function is a small block of defined reusable code that can take an input, changes it in some way and create desirable output.

Anatomy of a Function

def name( parameters ):
statements

  • def - Define - what does the function do?
  • name - name - the function semantically.
  • parameters - inputs - formal parameters.
  • statement - expression
  • return (tells Python exactly what the output should be).

The Return Statement exits the function. A return statement with no arguments is the same as a NONE.

Defining and Calling a Function

Simple Function without formal Parameters

>>> def hello():
... print("Hello World !")

>>> def say_hello(name):
... greeting = "Hello " + name + "!"
... return greeting
...
>>> print say_hello("Miriam")

>>> print say_hello("Andy")
Hello Andy!

Why use a Function

Functions once defined can be saved and reused, allowing the same operation to be executed over and over again, on whatever data is passed into it as arguments, great for repetitive tasks. Functions are a building block of programming problem solving strategy, a large problem can be broken up into smaller problems solved by functions. This is termed Function Composition, the process of combining simple functions to make more complex ones. The result of each function is passed as an argument in the next Function.

if statement

An if statement can be used to conditionally execute code, depending on whether or not the if statement’s condition is True or False.

elif statement

An elif (that is else if) statement can follow an if statement. Its block executes if its condition is True and all of the previous conditions have been False.

else statement

An else statement comes at the end. Its block is executed if all of the previous conditions have been False.

If the condition in an if statement is true the execution enters the indented code that follows.
If the condition is false the execution skips the indented code.
